# <script>, <script async>, <script defer>의 차이점은 무엇인가요?

3가지의 차이점은 스크립트 로딩 방식에 있습니다.

기본 <script> 태그는 HTML 파싱을 중단하고 즉시 로드 및 실행됩니다.
따라서 HTML이 렌더링되기 전에 스크립트가 실행되어 아직 존재하지 않는 DOM 요소를 조작하려고 할 수도 있습니다.

<script async> 태그는 HTML 파싱과 동시에 스크립트를 다운로드 하고 다운로드가 완료되는 즉시 실행합니다.
실행 중에는 HTML 파싱을 중단합니다.
따라서 스크립트가 실행되기 전에 모든 요소가 로드되어야 합니다.

<script defer> 태그는 HTML 파싱과 동시에 스크립트를 다운로드 하고 실행합니다.
async와 달리 HTML 파싱이 끝난 후 실행됩니다.
